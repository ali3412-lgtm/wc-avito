# Исправленные перечисления значений для Avito

## Проблема
В плагине не хватало правильных перечислений значений согласно документации Avito. Некоторые поля были текстовыми вместо выпадающих списков с фиксированными значениями. После пересмотра документации было выявлено, что поле "Тип услуги" (ServiceType) отсутствует в текущей официальной документации Avito и было удалено из плагина.

## Исправления

### 1. ServiceType (Тип услуги) - УДАЛЕНО
**Статус:** УДАЛЕНО из плагина  
**Причина:** Поле отсутствует в текущей официальной документации Avito

**Что было удалено:**
- Поле ServiceType из форм добавления и редактирования категорий
- Поле ServiceSubtype из форм добавления и редактирования категорий
- Соответствующие столбцы из CSV экспорта
- Соответствующие поля из XML генератора
- Обработка этих полей в функциях сохранения

### 2. Category (Категория)
**Было:** 3 категории  
**Стало:** 12 категорий

**Добавленные категории:**
- Предложение услуги (основное)
- Оборудование для бизнеса
- Недвижимость
- Транспорт
- Личные вещи
- Для дома и дачи
- Бытовая электроника
- Хобби и отдых
- Животные
- Готовый бизнес

### 3. ContactMethod (Способ связи)
**Статус:** Уже был корректным  
**Значения:** По телефону и в сообщениях, По телефону, В сообщениях

## Файлы, которые были изменены:

1. **`includes/product-fields.php`**
   - Удалены поля ServiceType и ServiceSubtype из форм
   - Расширен список категорий
   - Обновлены формы добавления и редактирования категорий

2. **`includes/csv-generator.php`**
   - Удалена колонка ServiceType из CSV заголовков
   - Удалены все ссылки на ServiceType из генераторов данных

3. **`includes/xml-generator.php`**
   - Удалены поля ServiceType и ServiceSubtype из XML генератора
   - Обновлена функция установки общих настроек

4. **`CSV_ATTRIBUTES_GUIDE.md`**
   - Обновлена документация с удалением ServiceType
   - Перенумерованы оставшиеся атрибуты

5. **`ENUMERATED_VALUES_FIX.md`**
   - Обновлен с информацией об удалении ServiceType
   - Исправлены типы полей с string на enum

## Соответствие документации Avito

Все изменения выполнены в соответствии с официальной документацией Avito:
- https://www.avito.ru/autoload/documentation/templates/123374

### Преимущества исправлений:

1. **Валидность данных** - Пользователи больше не могут ввести некорректные значения
2. **Соответствие API** - Все значения теперь точно соответствуют требованиям Avito
3. **Удобство использования** - Выпадающие списки проще в использовании чем текстовые поля
4. **Предотвращение ошибок** - Исключены опечатки и неправильные значения

### Обратная совместимость:

- Существующие данные остаются без изменений
- Если старое значение не соответствует новому списку, поле останется пустым
- Пользователь сможет выбрать правильное значение из списка при следующем редактировании

## Результат

Теперь плагин полностью соответствует документации Avito и предоставляет только валидные перечисления значений для всех полей, что гарантирует успешную загрузку объявлений на платформу.
